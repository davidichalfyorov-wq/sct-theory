# Formal Verification: Background Briefing

The SCT project uses Lean 4 with Mathlib4 and PhysLean to formally
verify key mathematical identities, algebraic relations, and structural
theorems. The goal is to provide machine-checked proofs for the
foundational results of the theory, complementing the numerical and
analytical verification layers.

## Infrastructure

**Lean 4 v4.28.0** with three verification backends:

1. **Local (PhysLean + Mathlib4).** The primary backend. PhysLean
   provides Lorentz group representations, Standard Model Lagrangian,
   and FLRW spacetime formalisations. Mathlib4 provides the
   mathematical library (real analysis, linear algebra, measure theory,
   topology). Accessed via `check_lean_local()`.

2. **Aristotle (cloud).** A cloud-based theorem prover that generates
   Lean 4 proofs from sorry statements. Used for difficult proofs
   that exceed local computation time. Proofs generated by Aristotle
   require `import Mathlib.Tactic`. Accessed via `prove()` and
   `prove_file()`.

3. **WSL SciLean.** The SciLean library for scientific computing in
   Lean 4, running under WSL2. Provides formalised numerical analysis,
   matrix algebra, and differential geometry. Accessed via
   `prove_scilean()`.

**Project structure.** The Lean project is located at
`theory/lean/SCTLean/` (with a junction at `C:\sct-lean`). It contains:
- 5 modules: Basic, FormFactors, SpectralAction, Tensors, StandardModel.
- 46 proof files (44 sorry-free, 2 with remaining sorry statements).
- 41 canonical identities registered in the SCT_IDENTITIES database.
- 105 CJ bridge theorems (sorry-free) across 7 proof files.
- 91 formal verification tests in the test suite.

## What has been formalised

### Algebraic identities (proven, sorry-free)

- alpha_C = 13/120 (total Weyl coefficient, from SM multiplicities).
- c_1/c_2 = -1/3 + 120(xi - 1/6)^2/13 (general formula).
- 3c_1 + c_2 = 6(xi - 1/6)^2 (scalar decoupling condition).
- V(0) = 0 (modified potential vanishes at origin: no singularity).
- BV canonical transformation (Barvinsky-Vilkovisky alpha-insertion
  technique, algebraic verification).
- Local limits: beta_W^{(s)} and beta_R^{(s)} for s = 0, 1/2, 1.
- SM multiplicities: N_s = 4, N_D = 45/2, N_v = 12.
- Effective masses: m_2^2 = (60/13) Lambda^2, m_0^2(xi=0) = 6 Lambda^2.
- Master function Taylor coefficients: a_n = (-1)^n n! / (2n+1)!.

### Structural theorems (proven, sorry-free)

- CL commutativity: Weierstrass M-test for N-pole fakeon convergence.
- GZ constancy: g_A = -13/60, Hadamard genus argument.
- KK resolution: quantum number mismatch excludes Kugo-Ojima quartet.
- CJ bridge theorems: 105 sorry-free theorems for the CJ estimator
  algebraic framework.

### Partial formalisations (sorry remaining)

- 2 proof files retain sorry statements (specific intermediate lemmas
  where the Lean proof term exceeds the timeout or requires manual
  construction).

## What has NOT been formalised

1. **Spectral positivity theorem.** The statement Pi_entire(z) > 0
   for all real z >= 0 has been verified numerically to z = 100 but
   not formalised in Lean. The proof would require formalising the
   Weierstrass factorisation of Pi_TT and proving positivity of the
   product, which involves transcendental functions (erfi, Gaussian
   integrals) that are not yet in Mathlib4.

2. **CHIRAL-Q chirality identity at higher loops.** The identity
   {D, gamma_5} = 0 has been formalised at one loop. Extension to
   arbitrary loop order requires formalising the BV-BRST cohomology
   and the block-diagonal structure of counterterms, which exceeds
   current PhysLean capabilities.

3. **NT-4b full nonlinear field equations.** The variational
   derivatives delta S_sp / delta g^{mu nu} = 0 have been derived
   but not formalised. This requires formalising the functional
   derivative on the space of metrics, which is a major
   infrastructure investment.

4. **Angular integral over S^2.** The integral int_{S^2} f(theta, phi)
   d Omega appears in the derivation of several form factor results.
   Only the rational skeleton (combinatorial identities involving
   binomial coefficients) has been formalised. The full angular
   integral with transcendental integrands is not in Mathlib4.
